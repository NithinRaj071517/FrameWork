OOPS (Object Oriented Programmimg):
			In simple words, it’s a way of writing programs by organizing code into objects instead of just functions and logic.
			Java is designed around OOPS principles, which help in creating modular, reusable, and maintainable code.
			OOPS makes object more powerful and robust .
			Used to improve coding standards and performance
			
			
		Four Pillars of OOPS:
			1. Abstraction - Hiding the implementation and expose only the behavior 
							 Abstraction in OOP means hiding the internal details of how something works and showing only the essential features.
				
				Achieved using:
					Interfaces:	
								An interface is a blueprint of a class. 
								An interface is a collection of abstract methods (methods without implementation) and constants.
								Purpose of interface is achieve Abstraction & Multiple Inheritance in java
								
							
						Key Points:	
								Used heavily in Seleinium.
								Achieves 100% abstraction.
								You cannot create an object of an interface.
								All methods in an interface are public and abstract by default
								Variables in an interface are public, static, and final by default.
								A class uses implements keyword to use an interface.
								A class can implement multiple interfaces (Java supports multiple inheritance through interfaces).
								Does not Supports Constructor and  variables
								
					Abstract class:
								Declared using abstract keyword.
								Can have atleast one abstract method.
								Can also have concrete method.
								Supports Constructor and  variables
					
						Key Points:	
							
							Achieves partial  abstraction.
							You cannot create an object of an interface.
							Supports Constructor and  variables
							A class uses extends keyword to use an abstract class.
						
							
							 
							 
				Where Used :
							Seleinium: Seleinium itself best example for abstraction.
										Webdriver, Webelement, Alert etc..
							Framework: Driver factory, base class
									   Step Definition & POM  is the best example for abstraction.			 
						 
							 
							 
							 
							 
							 
							 
							 
			2. Inheritance - It allows one class(Child /Sub class) to  acquires the properties and behaviours of  another class(Parent/Super). 
							 You are able to reuse more code.
							 Establishes a hierarchical relationship between classes.
							 Achieved by extends keyword
							
						Types:
							1. Single/Simple - a class inherits from only one superclass. This means a child class (subclass) can extend only one parent class at a time.
									Class A(Parent/Super)  --> Class B (Child/Sub class)
									Class B extends Class A 
									(*)We can access parent class methods & properties in child class using child class object.
									
									
							2. Multilevel - Multilevel inheritance means a class inherits from a class that itself inherits from another class—forming a chain.
									Grandfather --> Parent --> Child
							
							
							
							3. Multiple (Java does NOT support multiple inheritance with classes (to avoid ambiguity), but it supports it through interfaces.)
								
								Class  --> Class = Extends
								Class  --> Interface = Not Possible
								Interface --> Class = Implements
								Interface --> Interface = Extends
								
					Where Used :
							Seleinium: Browser setup code, baseclasses, resusable methods
							Framework: Common framework structure
									  Common actions base page and specific page logic in child page
									   POM  is the best example for Inheritance.			 		
							 
							 
							 
			3. Polymorphism - One object, many implementations 
							  In Java the same method name behaves differently based on parameters,Object type 
							
							
							Types:
								1. Static Binding / Compile Time Polymorphism
									Achieved by Method Overloading
									Happen within class
									Creating same method is not allowed , but able to achieve by increasing no.of arguments or by changing the type of arguments
								2. Dynamic Binding / Run Time Polymorphism
									Achieved by Method Overriding
									Method in parent class , reimplemented in child class
									Happen within the inherited class
									Achieved using Inheritance
						
						Where Used :
							Seleinium: Run Time Polymorphism
										Webdriver - Parent
										ChromeDriver - Child 
									   Compile  Time Polymorphism
										Actions 
										Alerts
										
										
							Framework: Cross browser testing
									   Step Definition & POM  is the best example for abstraction.								
									
			4. Encapsulation - Wrapping data(variables) and methods together in a class 
							   Restricting direct access to data
							 
							 Key Points:
								The data inside a class is kept private.
								Access to that data is given through public getter and setter methods.
								This helps protect data from unauthorized access or modification.
							 
							 Where Used :
								Seleinium: Encapsulation in Seleinium used to hide Webelements and expose only user actions through method.
								Framework: It improves maintainablity, security and reduce impact when UI changes occurs.
										   Page Object Model is the best example for encapsulation.

Collections:
		Collection is a concept of storing multiple data into a single variable	based on some condition.
		Array is size fixed while creation whereas collection may grows or shrinks .
		Collection is a dynamic in size.
		
		Java Collections Framework provides a set of interfaces (like List, Set, and Map) and 
												a set of classes (ArrayList, HashSet, HashMap, etc.) that implement those interfaces.
		
		Example: Think of the Collections Framework as a toolbox.
			Interfaces like List define what tools can do, and classes like ArrayList are the actual tools that do the work.
			
			
			1. List
				List is a single dimensional collection.
				List is a Interface
				Props:
					1. List maintains the insertion order 
					2. List allow duplicate values 
						
				List is a single dimensional collection.
				List is a Interface
					ArrayList:	
							List -> Interface
							ArrayList -> Implementation class (Predefined / Library class)
							In the generic we are not allowed to store primitive datatypes 
				
						-- list.add(value) -->  it will add value in arrayList
						--list.add(index,value) --> it will add the value in the index place
						--list.set(index,value) --> it replace the value in the index place
						list.size() --> Provides size of the array
						--list.remove(value) --> removes the value 
						-- list.get(index)  --> Provides the value of the index
						list.contains(value) --> verify values present in List
						--list.clear() --> Remove all the values from the list
						list.isEmpty() --> Provide true or false if list is empty
						Collections.sort(list) --> Order the list in ascending order
						-- list.indexOf(value) --> Getting the index of the value
				
				
			2. Set
				Set is a single dimensional collection.
				Set is also an interface
				Props:
					1. Set may or may not maintains the insertion order 
					2. Set do not allow duplicate values 
					
				Common classes that implement Set:
					1. HashSet- Random order as output
					2. TreeSet - Ordered output
					3. LinkedHashSet - Maintain insertion order 
					
				No update, get ,insert,remove supports	
					
				.addAll() --> To add the set/list to list/set
					List -> Set
					List -> List 
					Set-> Set
					Set-> List
					
				Want to iterate over the set

					2. Convert the set into list , then iterate over the list
					
			3. Map( Table of data)
				Map is a two dimensional collection.
				Map is also an interface
				Map represents a collection of key-value pairs.
					Entry --> <Key,Value>
					<Key,Value>  --> Pair of data
					Key --> Set (do not allow duplicate values )
					Value --> List (allow duplicate values )
					
					Map is a collection of Entry  Map--> Collection<Entry>

					map.put(key,value) --> To add values

Static:
	Static means “belonging to the class, not to instances.”
	static member is created once per class not per object
	Memory is allocated only once when the class is loaded.
	Can access static members via the class name ,no object creation is needed to access static members
	Static value cannot change , non-static value can change.
	Static store in Stack/ Class level memory where as non-static stores in heap memory
	
	Important Points:
		Static variable can access in any method without using any object .
		A non static method can access another non-static method without using any object
		A non-Static variable/not defined globally  can access only within that  method without using any object.
		Static methods and variables can’t access non-static members directly.
		
		
	
			
	
			Differnce between Static vs Non-Static Members:	
					Static Belongs To	Class (shared by all objects) where as 	Non-Static Belongs To Individual object (unique per instance)
					Static Memory Allocation Created once when class is loaded where as Non-Static Memory Allocation Created separately for each object
					Static can	access using class name where as Non-Static can	access  using object reference
					Static Can Access only other static members directly	where as Non-Static Both static and non-static members
					Static Common Use	Utility methods, constants, shared data	where as Non-Static Object-specific data and behavior
	
	
	Static can be used in places :
		1. Static Variable  (Class variable) -- shared among all instances of the class 
												value of the variable remains same everywhere 
												Static variable can access with class name & not with object 
		2. Method level  -- belongs to the class rather than to any object.
							Static method can access with class name & not with object and methods directly
							method cannot be overridden(i.e cannot be used in another class)
							Cannot use "this" & "super" keywords
	
	
	
Final:
	Final is a modifier you can apply to variables, methods, and classes to prevent further modification or extension


	Final can be used in places :
		1. Class level --> If it is final class it cannot be inherited.
		2. Method level  --> If it is final method it cannot be overridden.
		3. Variable level  --> If it is final variable it cannot be changed.
		
		
	Common confusions
		final vs finally vs finalize:	
			final: modifier preventing change/inheritance (this note).
			finally: try/catch block that always executes (for cleanup).
			finalize(): legacy GC hook; deprecated/removed in modern Java.	
		
	
	
	this --> used to call the current class reference or method
	super --> used to call the parent class reference or method
